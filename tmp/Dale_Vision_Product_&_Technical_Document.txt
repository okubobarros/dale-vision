DaleVision
Documento Técnico de Produto (Arquitetura, Dados, IA, Escala)
1. Visão Técnica
O DaleVision é uma plataforma edge-first de inteligência operacional baseada em:
Computer Vision (CV)
Event Streaming (stream + batch)
Analytics operacional (features/KPIs)
Copiloto (regras + LLM)
O sistema é orientado a eventos: vídeo → eventos → decisão → ação.
2. Arquitetura Geral
Camadas
Edge (processamento local)
Ingestão de Eventos (API idempotente)
Pipeline de Dados (stream + batch)
Backend Multi-tenant (serviços + RBAC)
Analytics (features/KPIs)
Copiloto (regras + LLM)
Integrações (PDV, WhatsApp, Email)
Fluxo
Câmera/NVR → Edge → Eventos → Ingest API → Fila → Workers → DB → KPIs → Alertas/Copiloto

10. ROI Engine (Dinheiro em Risco)
Fórmulas
conversion_rate = transactions / flow_in
lost_sales = max(0, queue_wait_time - threshold) * abandonment_rate * 
avg_ticket
idle_cost = idle_minutes * (custo_hora/60)
money_at_risk = lost_sales + idle_cost
Inputs
CV: fila, fluxo, dwell
PDV: vendas, ticket
Parâmetros: thresholds, custo
Outputs
R$ em risco (tempo real)
R$ recuperável (ação sugerida)

17.4 ROI Financeiro (como calcular)
Antes

conversão: 12%
Depois
fila média: 3 pessoas
conversão: 18%
Cálculo
Δ conversão = +6%
receita incremental = flow_in * Δ conversão * ticket médio

23.3 Critérios de aceite por indicador
queue_length
erro médio absoluto ≤ 1 pessoa em cenários validados
estabilidade: não oscilar sem mudança real
flow_in / flow_out
erro relativo ≤ 10% por hora
robusto a iluminação padrão da loja
occupancy
erro relativo ≤ 15% por janela
coerência entre câmeras adjacentes
queue_wait_time
erro ≤ 10 segundos em amostra anotada
consistência entre janelas consecutivas
conversion_rate
diferença máxima aceitável definida por reconciliação PDV
sem cálculo quando qualidade do fluxo estiver abaixo do threshold
money_at_risk
auditável: cada valor precisa ser rastreável aos inputs
parâmetros de fórmula versionados
