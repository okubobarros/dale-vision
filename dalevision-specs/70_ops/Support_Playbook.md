# Support Playbook

## Objetivo
Padronizar suporte e reduzir tempo de resolução.

## Fluxo de atendimento
1. Triagem
2. Coleta de dados
3. Diagnóstico
4. Resolução
5. Follow-up

## Dados mínimos
- Store ID
- Org ID
- Logs do edge-agent
- Erro no backend

## Coleta rápida (Edge Agent)
- `Diagnose.bat` (gera ZIP de diagnóstico)
- `logs\agent.log`
- `logs\update.log`

## Escalonamento
- Quando envolver infra, billing ou segurança
